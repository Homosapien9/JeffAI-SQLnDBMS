Core Definitions — Board 2008–2020
Q1 What is a database? How is it different from a DBMS?
Board 20122M
▼
Answer

Database: An organized collection of structured data or information stored and accessed electronically. It stores data in tables (relations) with rows and columns.

DBMS (Database Management System): Software that manages, stores, retrieves, and manipulates data in a database. Examples: MySQL, Oracle, MS Access.

Difference: A database is the data; DBMS is the software that creates and manages that data.

Q2 Define the following terms with respect to a relational database:
(a) Tuple   (b) Attribute   (c) Domain   (d) Degree   (e) Cardinality
Board 2015–20193M
▼
Answer
  • Tuple: A single row in a relation (table). Represents one record/entity. Example: One student's row in a STUDENT table.
  • Attribute: A column in a relation. Represents a property of an entity. Example: Name, RollNo.
  • Domain: The set of all possible values an attribute can take. Example: Domain of Marks = {0, 1, 2, …, 100}.
  • Degree: The total number of attributes (columns) in a relation/table.
  • Cardinality: The total number of tuples (rows) in a relation/table.
Q3 A table PRODUCTS has 5 columns and 8 rows. What is its degree and cardinality?
Board 2018, SQP 20221M
▼
Answer

Degree = 5 (number of columns/attributes).

Cardinality = 8 (number of rows/tuples).

Q4 If table A has 6 rows and 3 columns, and table B has 5 rows and 2 columns, what will be the Cartesian product of A and B?
Board 20251MMCQ
▼
Answer

Cartesian Product of A × B:

  • Rows = 6 × 5 = 30 rows
  • Columns = 3 + 2 = 5 columns

The Cartesian product combines every row of A with every row of B, and all attributes from both tables appear as columns.

Keys & Constraints — Board 2008–2025
Q5 Define Primary Key. Give an example.
Board 20081M
▼
Answer

A Primary Key is an attribute (or set of attributes) that uniquely identifies each row in a table. It cannot have NULL values and cannot have duplicate values.

Example: In a STUDENT table, RollNo is the primary key — every student has a unique roll number.

Q8 What is a Foreign Key? How does it establish a relationship between two tables?
Board 2014, 20222M
▼
Answer

A Foreign Key is an attribute in one table that refers to the Primary Key of another table. It establishes and enforces a referential integrity link between the two tables.

Example: In a ORDERS table, CustomerID is a foreign key that references CustomerID (primary key) in the CUSTOMERS table. This ensures that no order can be placed for a non-existent customer.

H1 Consider table STUDENT(RollNo, Name, Marks, Grade, City). Predict the output of:
SELECT Grade, COUNT(*), MAX(Marks) FROM STUDENT GROUP BY Grade HAVING COUNT(*)>2;
HOTS2M
▼
Answer

This query groups students by Grade, then counts how many students are in each grade group and finds the maximum marks in each group. The HAVING clause filters out groups with 2 or fewer students — only grades with more than 2 students appear in the output.

Key points to mention:

  • GROUP BY Grade → creates one row per unique grade
  • COUNT(*) → number of students in that grade
  • MAX(Marks) → highest marks in that grade
  • HAVING COUNT(*) > 2 → only grades with 3+ students shown

H3 A student wrote: SELECT * FROM EMPLOYEE WHERE SALARY > AVG(SALARY);
Will this query execute successfully? If not, correct it.
HOTS2M
▼
Answer

No, this query will NOT execute successfully.

Reason: Aggregate functions like AVG() cannot be used directly inside a WHERE clause. WHERE filters individual rows before any grouping/aggregation occurs.

Correct query using a subquery:

SELECT * FROM EMPLOYEE
WHERE SALARY > (SELECT AVG(SALARY) FROM EMPLOYEE);

The inner subquery first calculates the average salary, and the outer query uses that result in the WHERE clause.

H4 Predict the output of these SQL commands on STUDENT(RollNo, Name, Class, Marks):
(i) SELECT Name FROM STUDENT WHERE Name LIKE '_a%';
(ii) SELECT Name FROM STUDENT WHERE Name LIKE '%a_';
HOTS2M
▼
Answer

(i) WHERE Name LIKE '_a%'

_ matches exactly one character; a is the second character; % matches zero or more remaining characters.

→ Displays names where the second character is 'a'. Example: Rahul, Sanjay, Rajiv.

(ii) WHERE Name LIKE '%a_'

% matches zero or more chars; a is second-to-last character; _ matches exactly one final character.

→ Displays names where the second-to-last character is 'a'. Example: Sonal, Komal, Rajas.

H5 A teacher has a table MARKS(RollNo, Subject, Marks). She runs:
SELECT RollNo, SUM(Marks) FROM MARKS;
Will this work? Explain what is wrong and write the corrected query.
HOTS2M
▼
Answer

Problem: In SQL, when you use an aggregate function (SUM), all other non-aggregated columns in the SELECT must appear in the GROUP BY clause. RollNo is not in GROUP BY, which causes an error in strict SQL mode.

Corrected query (total marks per student):

SELECT RollNo, SUM(Marks) AS TotalMarks
FROM MARKS
GROUP BY RollNo;

S8 Consider the table EMPLOYEE(EID, ENAME, DEPT, SALARY, CITY).
Write SQL for:
(i) Display names and department of employees earning more than 40000, ordered by department.
(ii) Display the number of employees in each city.
(iii) Display departments where total salary exceeds 200000.
SQP 2025-26SQP3M
▼
Answer

(i)

SELECT ENAME, DEPT FROM EMPLOYEE
WHERE SALARY > 40000
ORDER BY DEPT;

(ii)

SELECT CITY, COUNT(*) AS NumEmployees
FROM EMPLOYEE
GROUP BY CITY;

(iii)

SELECT DEPT, SUM(SALARY)
FROM EMPLOYEE
GROUP BY DEPT
HAVING SUM(SALARY) > 200000;
